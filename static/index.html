Schema & Architecture

How multi-tenant data isolation works

Isolation model

Every table carries an org_id foreign key. The authenticated employee's org_id is baked into their JWT — the server never trusts a client-supplied header. All queries are automatically scoped with WHERE org_id = :org_id.

Role-based access control is also JWT-driven: managers and above can create, edit, and delete employees and approve or reject leave requests. Junior roles are read-only.
